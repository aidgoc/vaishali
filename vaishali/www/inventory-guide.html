1. Overview

DSPL manages 6,502 items across 14 warehouses — from raw electronic components (BK*/BL* coded items) to finished crane safety products (ACD, DRM-3400, DJ-1005, E-DASH, F-Dash, WWSI, MRT, TPS). Accurate stock levels are critical because:

  • Production depends on knowing what raw materials are available
  • Sales needs projected quantity to promise delivery dates
  • Finance requires accurate valuation for balance sheet and costing
  • Safety-critical products cannot ship with missing or wrong components
Every stock movement must go through a Stock Entry in ERPNext. Never move physical stock without a corresponding system entry — it creates phantom variances that compound over time.

Stock moves in ERPNext are recorded through Stock Entries, each with a specific purpose type. The system uses FIFO (First In, First Out) valuation by default.

2. Process flow

Goods arrive from supplier | v [Material Receipt] → Stores-DSPL (increases stock) | v Production needs raw materials | v [Material Transfer] → Stores-DSPL to WIP-DSPL | v [Manufacture] → Consume raw materials, produce finished goods | (from Work Order) v Finished goods move to FG warehouse | v [Material Transfer] → WIP-DSPL to Finished Goods-DSPL | v Customer order fulfilled | v [Material Issue] or Delivery Note → Finished Goods-DSPL (decreases stock) | v Monthly / quarterly: [Stock Reconciliation] to match physical count

3. Stock Entry types

Material Receipt

Purpose: Goods received from supplier  |  Effect: Increases stock
  1. Go to Stock > Stock Entry > New
  2. Set Purpose to Material Receipt
  3. Set the Target Warehouse (usually Stores-DSPL)
  4. Add items: item code, quantity received, basic rate
  5. Attach supplier delivery challan or invoice reference
  6. Click Save, then Submit
If you have a Purchase Order, create the Material Receipt from the Purchase Receipt instead — it auto-fills items, rates, and links back to the PO.

Material Issue

Purpose: Consume stock (non-production)  |  Effect: Decreases stock
  1. Go to Stock > Stock Entry > New
  2. Set Purpose to Material Issue
  3. Set the Source Warehouse (where items are stored)
  4. Add items and quantities to be consumed
  5. Add a reason in remarks (e.g., "Service kit for WC-00045")
  6. Click Save, then Submit
Check the projected quantity before issuing. If available qty is insufficient, the system will block the submission.

Material Transfer

Purpose: Move stock between warehouses  |  Effect: Neutral (source decreases, target increases)
  1. Go to Stock > Stock Entry > New
  2. Set Purpose to Material Transfer
  3. Set the Source Warehouse and Target Warehouse
  4. Add items and quantities to transfer
  5. Click Save, then Submit
Common transfers: Stores-DSPL to WIP-DSPL (production start), WIP-DSPL to Finished Goods-DSPL (production complete without Work Order).

Manufacture

Purpose: Consume raw materials, produce finished goods  |  Effect: Source items decrease, target item increases
  1. Create a Work Order for the item to be manufactured (links to BOM)
  2. From the Work Order, click Start to create a Stock Entry
  3. The system auto-fills:
    • Source items from the BOM (raw materials from Stores-DSPL)
    • Target item (finished good to Finished Goods-DSPL)
    • Quantities per the BOM
  4. Verify quantities and click Submit
Always use "from Work Order" for manufacture entries. Do not create standalone manufacture Stock Entries — they bypass BOM validation and cost tracking.

Repack

Purpose: Convert one item to another (repackaging, kitting)  |  Effect: Source item decreases, target item increases
  1. Go to Stock > Stock Entry > New
  2. Set Purpose to Repack
  3. Add source items (what you are consuming) with the Source Warehouse
  4. Add target items (what you are creating) with the Target Warehouse
  5. Set quantities for both source and target
  6. Click Save, then Submit
Use Repack when you are repackaging bulk items into smaller units or combining items into a kit without a formal BOM.

4. Stock Reconciliation

Adjust stock to match physical count

Who: Store Manager / Accounts  |  When: Monthly or quarterly physical count
  1. Conduct a physical stock count of the warehouse
  2. Go to Stock > Stock Reconciliation > New
  3. Select the Company (Dynamic Servitech Private Limited)
  4. Set the Posting Date and Posting Time
  5. Add items with their:
    • Item Code
    • Warehouse
    • Quantity (the actual physical count)
    • Valuation Rate (current rate per unit)
  6. The system shows the difference between system qty and entered qty
  7. Click Save, then Submit to adjust
Stock Reconciliation creates automatic stock adjustments. Review differences carefully before submitting — large variances should be investigated, not blindly reconciled.
Schedule reconciliation: monthly for high-value items (finished goods, PCBs), quarterly for low-value items (fasteners, cables, packaging).

5. Warehouses

WarehousePurposeWhat goes here
Stores - DSPLMain receiving warehouseAll incoming raw materials and components from suppliers
Finished Goods - DSPLFinished goods storageCompleted products ready for dispatch (ACD, DRM, E-DASH, etc.)
Work In Progress - DSPLManufacturing floorItems currently being assembled or tested

Additional warehouses exist for specific purposes (service stock, rejected items, transit, etc.). All warehouse names must end with " - DSPL" to comply with ERPNext's company-suffix naming convention.

To create a new warehouse: Stock > Warehouse > New. Set the parent warehouse group and ensure the name ends with " - DSPL". Only System Managers can create warehouses.

6. PWA access

Field app stock view

Where: dgoc.logstop.com/field  |  Route: #/stock

The field PWA provides a mobile-friendly stock view at #/stock. Field staff can:

  • Search items by name or item code (BK*/BL* codes)
  • View current stock across warehouses
  • Check projected quantity (includes reserved and ordered)

Access is role-based — field engineers see stock levels but cannot create Stock Entries from the PWA. Stock Entries must be created from the desk (dgoc.logstop.com/app).

7. Dashboards

Stock monitoring

Where: DSPL Operations workspace  |  View: stock_levels
  • DSPL Operations workspace: Number Cards showing stock summary, pending receipts, and pending issues
  • Stock Levels view: Available in the production dashboard — shows item-wise stock across all warehouses
  • Bin Report: Stock > Bin — shows actual qty, planned qty, reserved qty, and projected qty per item per warehouse
The Bin report is the most detailed view. Use it to check why projected quantity differs from actual quantity — it shows reserved (for Sales Orders), ordered (from Purchase Orders), and planned (from Work Orders) quantities.

9. Tips and best practices

  • Always use "from Work Order" for manufacture entries — it ensures BOM quantities are followed and costs are tracked correctly
  • Check projected qty before issuing — projected qty accounts for reserved (SO), ordered (PO), and planned (WO) quantities
  • Reconcile monthly — at minimum, reconcile finished goods and high-value components monthly
  • FIFO valuation — DSPL uses First In, First Out. The oldest stock is consumed first for costing purposes
  • Attach references — always link Stock Entries to source documents (Purchase Receipt, Work Order, Delivery Note) for traceability
  • Rejected items — move rejected incoming goods to a separate warehouse (Rejected - DSPL), never mix with good stock
  • Minimum stock levels — set reorder levels on critical items (PCBs, display units) so the system alerts when stock runs low
  • Batch tracking — for components where supplier batch matters (ICs, sensors), enable batch tracking on the Item master

1. Overview & scope

This SOP standardises material handling at the DCEPL ERS Central Workshop store — covering Material Inward, Material Issue, and Material Return — to ensure proper control, traceability, and accuracy of inventory.

Signed off 25 March 2026 by Anirudha Bhide (prepared) and Ramdas Avhad (reviewed).

ActivityResponsibility

Scope is DCEPL only. The system gates fire only when company is "Dynamic Crane Engineers Private Limited" AND the warehouse is in site_config.dcepl_ers_warehouses (default: Stores - DCEPL). DSPL, Gear Up, and other DCEPL warehouses are unaffected.

2. Material Inward (Purchase Receipt & QA/QC)

Every inward movement at the CWS store must pass a Quality Inspection before the Purchase Receipt can be submitted. The system blocks PR submission if any item lands in Stores - DCEPL without an Accepted QI.

Inward flow

Who: Store Executive + Maintenance Head
  1. Material arrives at the gate. Gate guard logs inward register entry; verifies invoice/challan present.
  2. Material is moved to the store with required documents.
  3. Store Executive creates a Purchase Receipt (or pulls from PO) in the desk.
  4. For each row landing in a CWS warehouse, create a Quality Inspection:
    • Use template "DCEPL ERS Store QA/QC"
    • 5 parameters — tick "Yes" on each: Inv OK, PO Match, Qty Match, Physical OK, Technical OK
    • Status auto-flips to Accepted/Rejected based on the readings
  5. Submit the QI(s), then submit the Purchase Receipt — the system verifies every CWS-bound row has an Accepted QI.
  6. Store Executive assigns bin location, updates bin card, and closes the process.
If the material fails inspection, mark the QI Rejected and route to Purchase for return-to-vendor or RTO. Do NOT submit the PR until rejected items are reversed off it.
The 5 QI parameters use formula-based criteria (reading_1 == "Yes"). Anything other than the literal "Yes" fails the inspection. Don't bypass by editing the formula — if a parameter doesn't apply to a particular item, raise it to Anirudha to amend the SOP and the template together.

Filter Items in the desk by ERS ABC Class, Stock Status, ABC Segment, or ERS Department — all four are standard filters on the Item list.

4. Stock monitoring & re-order

Stock is monitored using three colour bands. Each item has a defined Min / Reorder / Max level on its Item master — the same numbers as the SOP master sheet.

BandTriggerAction
GREENStock above Reorder levelContinue monitoring
YELLOWStock at or below Reorder, above MinPlan replenishment within 15 days
REDStock at or below MinAuto-Material Request fires immediately

Auto Material Request

Who: System (cron)  |  What: ERPNext reorder_item scheduler

With Stock Settings > Auto Indent = enabled, the daily reorder cron creates a Purchase Material Request whenever actual qty in Stores - DCEPL falls below the Min level on a configured item.

  • 405 items have reorder rows configured against Stores - DCEPL as of the first import.
  • 468 of the 873 matched parts are in the AsPerRequirement segment with no Min/Reorder data — they're skipped (intentional: don't auto-MR for one-off parts).
  • Material Requests created by the cron land in Draft. Approve them as you would any MR.
To see what's about to trigger an auto-MR, open Stock > Bin and filter Warehouse = Stores - DCEPL, sort by Actual Qty ascending. Compare against the Reorder Level column on the Item.

5. Material Issue & MRN approval

Every Material Issue out of Stores - DCEPL requires an originating Material Requisition Note (Material Request, type "Material Issue") that has been approved by an HOD. The Stock Entry refuses to submit otherwise.

Site Technician flow

  1. Site tech raises a Material Request (type Material Issue, set warehouse Stores - DCEPL).
  2. HOD opens the MR, reviews items / qty, ticks MRN HOD Approved, saves. The system stamps Approved By + Approved At automatically.
  3. Store Executive creates a Stock Entry (type Material Issue) referencing the approved MR for each row.
  4. On submit, the system verifies each source row's MR is approved. If not, submission is blocked with the list of unapproved MRs.
  5. Store updates bin card, hands material to the technician, and closes the loop.

Central Workshop (CSW) flow

  1. CSW raises an MR with required approvals (same flow as Site Tech).
  2. HOD approves on the desk by ticking the box.
  3. Store issues material via Stock Entry; bin card updated.

Sales Department flow

  1. Sales raises an MR for material going to a customer.
  2. PO and invoice details are verified via email and attached to the MR.
  3. HOD approves; Store packages and dispatches.
  4. Stock Entry is submitted referencing the approved MR. Outward Gate Pass is filled at the gate (see §7).
Only users with the DCEPL ERS HOD role can tick "MRN HOD Approved". Trying to tick the box without the role is rejected with a clear error. Until HOD users are mapped to that role, no Material Issue can submit out of Stores - DCEPL.

6. Material Return

  1. Returned material arrives at the store with delivery challan; gate inward stamp verified.
  2. Physical condition inspected for damage and usability.
  3. If usable: Material is re-stored, Stock Entry (type Material Receipt) brings it back into Stores - DCEPL, bin card updated.
  4. If damaged: damage is documented; HOD informed via email or WhatsApp; rejected stock moved to a dedicated rejected location (do NOT mix with good stock).
For tools and demo equipment that were issued under the Returnable flag of an Outward Gate Pass, click Mark Returned on the original Gate Pass when the item comes back. This stamps the return time and closes the loop.

7. Outward Gate Pass

Every outward movement at the workshop gate must be logged on a submittable Outward Gate Pass document (naming: GP-YYYY-#####).

Filling the Gate Pass

Who: Gate guard  |  Where: dgoc.logstop.com/app/outward-gate-pass/new
  1. Date and time auto-fill to now. Set Vehicle No and capture driver name + phone.
  2. Set Source Warehouse (default Stores - DCEPL) and Destination Type (Site / CSW / Customer / Vendor / Internal / Other) with the destination name.
  3. Optional: link the carrier as an Employee record.
  4. If material is expected back (tools, demo equipment), tick Returnable and set Expected Return Date.
  5. Optional: link the source document — Stock Entry, Delivery Note, or Material Request — for reconciliation. Always link the originating MRN where possible.
  6. Add items: item code, qty, UOM, serial/batch where applicable.
  7. Save and Submit. Status flips to Out.

Closing returnable Gate Passes

When a returnable item comes back:

  1. Open the original Gate Pass (filter list view by Status = Out, Returnable = ticked).
  2. Click Mark Returned in the toolbar.
  3. Status flips to Returned and the system stamps Returned At.
The list view colour-codes status: red Draft, orange Out, green Returned, blue Closed. Use the Status standard filter for daily reconciliation against Stock Entries.

9. Configuration & kill switches

All controls live in site_config.json and can be toggled per-site without code changes.

KeyDefaultEffect
dcepl_ers_warehouses["Stores - DCEPL"]Which warehouses are in scope for QA/QC, MRN, and Outward Gate Pass enforcement
dcepl_ers_qa_qc_enabledtrueSet false to disable Purchase Receipt QI guard
dcepl_mrn_approval_enabledtrueSet false to disable both MRN approval validation and Stock Entry guard
Stock Settings > Auto IndentenabledERPNext-native flag — turn off to stop auto-Material-Request creation
Re-running the importers is always safe (idempotent): vaishali.setup_ers_abc.import_abc writes only diffs; vaishali.setup_ers_reorder.import_reorder only adds rows that aren't already there. Run both after the store team adds new Items or fills missing Min/Reorder data.

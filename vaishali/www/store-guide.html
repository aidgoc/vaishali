1. Overview

This SOP standardises material handling at the DCEPL ERS Central Workshop store — covering Material Inward, Material Issue, and Material Return — so inventory stays accurate, traceable, and under control.

2. Material Inward (Purchase Receipt & QA/QC)

Every inward movement at the CWS store must pass a Quality Inspection before the Purchase Receipt can be submitted. The system blocks Purchase Receipt submission if any item is going into the CWS store without an Accepted Quality Inspection.

Inward flow

Who: Store Executive + Maintenance Head
  1. Material arrives at the gate. Gate guard logs the inward register entry; verifies invoice/challan are present.
  2. Material is moved to the store with the required documents.
  3. Store Executive creates a Purchase Receipt (or pulls one from a Purchase Order) on the desk.
  4. For each row going into the CWS store, create a Quality Inspection:
    • Use template "DCEPL ERS Store QA/QC"
    • 5 checks — tick "Yes" on each: Invoice OK, PO Match, Quantity Match, Physical OK, Technical OK
    • Status flips to Accepted automatically when all 5 are Yes
  5. Submit the Quality Inspection, then submit the Purchase Receipt — the system verifies every row has an Accepted inspection.
  6. Store Executive assigns the bin location, updates the bin card, and closes the process.
If the material fails inspection, mark the Quality Inspection Rejected and route to Purchase for return-to-vendor. Don't submit the Purchase Receipt until rejected items are removed from it.
All 5 checks must be answered "Yes" for the material to be accepted. If a check doesn't apply to a particular item, raise it to Anirudha so the SOP and the inspection template can be amended together — don't skip checks.

Filter Items in the desk by ABC Class, Stock Status, ABC Segment, or ERS Department — all four are available as standard filters on the Item list.

4. Stock monitoring & re-order

Stock is monitored using three colour bands. Each item has a defined Min / Reorder / Max level on its Item record — the same numbers as the SOP master sheet.

BandTriggerAction
GREENStock above Reorder levelContinue monitoring
YELLOWStock at or below Reorder, above MinPlan replenishment within 15 days
REDStock at or below MinAuto Material Request fires immediately

Auto Material Request

Who: System  |  When: Automatically, daily

The system automatically creates a Purchase Material Request whenever actual quantity in the CWS store falls below the Min level on a configured item.

  • 405 items have re-order rules configured today.
  • 468 of the 873 matched parts are in the AsPerRequirement segment with no Min/Reorder data — the system intentionally does not auto-MR for one-off parts.
  • Auto-created Material Requests land in Draft. Approve them as you would any normal MR.
To see what's about to trigger an auto Material Request, open Stock > Bin and filter by Warehouse equals the CWS store, then sort by Actual Qty ascending. Compare against the Reorder Level shown on the Item.

5. Material Issue & MRN approval

Every Material Issue out of the CWS store requires an originating Material Requisition Note (a Material Request, type "Material Issue") that has been approved by the HOD. The system refuses to issue stock otherwise.

Site Technician flow

  1. Site technician raises a Material Request (type Material Issue, set warehouse to the CWS store).
  2. HOD opens the MR, reviews items and quantity, ticks MRN HOD Approved, saves. The system stamps Approved By and Approved At automatically.
  3. Store Executive creates a Stock Entry (type Material Issue) referencing the approved MR for each row.
  4. On submit, the system verifies each source row's MR is approved. If not, submission is blocked with the list of unapproved MRs shown.
  5. Store updates the bin card, hands material to the technician, and closes the loop.

Central Workshop (CSW) flow

  1. CSW raises an MR with required approvals (same flow as Site Tech).
  2. HOD approves on the desk by ticking the box.
  3. Store issues material via Stock Entry; bin card updated.

Sales Department flow

  1. Sales raises an MR for material going to a customer.
  2. PO and invoice details are verified via email and attached to the MR.
  3. HOD approves; Store packages and dispatches.
  4. Stock Entry is submitted referencing the approved MR. Outward Gate Pass is filled at the gate (see §7).
Only users with the ERS HOD role can tick "MRN HOD Approved". Trying to tick the box without the role is rejected with a clear error.

6. Material Return

  1. Returned material arrives at the store with a delivery challan; the gate inward stamp is verified.
  2. Physical condition is inspected for damage and usability.
  3. If usable: the material is re-stored, a Stock Entry (type Material Receipt) brings it back into the CWS store, and the bin card is updated.
  4. If damaged: the damage is documented, HOD is informed via email or WhatsApp, and rejected stock is moved to a dedicated rejected location (don't mix with good stock).
For tools and demo equipment that were issued under the Returnable flag of an Outward Gate Pass, click Mark Returned on the original Gate Pass when the item comes back. This stamps the return time and closes the loop.

7. Outward Gate Pass

Every outward movement at the workshop gate must be logged as an Outward Gate Pass on the desk.

Filling the Gate Pass

Who: Gate guard
  1. Open the desk, search for "Outward Gate Pass", click New.
  2. Date and time auto-fill to now. Set Vehicle No and capture driver name + phone.
  3. Set Source Warehouse (defaults to the CWS store) and Destination Type (Site / CSW / Customer / Vendor / Internal / Other) with the destination name.
  4. Optional: link the carrier as an Employee record.
  5. If material is expected back (tools, demo equipment), tick Returnable and set Expected Return Date.
  6. Optional: link the source document — the Stock Entry, Delivery Note, or Material Request — for reconciliation. Always link the originating MRN where possible.
  7. Add items: item code, qty, UOM, serial/batch where applicable.
  8. Save and Submit. Status flips to Out.

Closing returnable Gate Passes

When a returnable item comes back:

  1. Open the original Gate Pass (filter the list view by Status = Out, Returnable = ticked).
  2. Click Mark Returned in the toolbar.
  3. Status flips to Returned and the system stamps Returned At.
The list view colour-codes status: red Draft, orange Out, green Returned, blue Closed. Use the Status standard filter for daily reconciliation against Stock Entries and Delivery Notes.
